DashUI

Template-basiertes HTML5 Framework für Smarthome Visualisierung

DashUI ist eine webbasierte Visualisierung für die HomeMatic CCU mit uneingeschränkter Gestaltungsfreiheit. Anzeige- und Bedienelemente (sogenannte "Widgets") können frei auf beliebigen Ansichten ("Views") platziert werden, außerdem können die Style-Eigenschaften sowie individuelle Attribute von Views und Widgets konfiguriert werden.

Die Gestaltung von Oberflächen kann auf verschieden Ebenen innerhalb von DashUI stattfinden:

  • Wer möchte muss sich nicht mit komplizierter Technik auseinandersetzen, sondern kann sich darauf beschränken vorhandene Widgets zu konfigurieren und im wesentlichen alles mit der Maus zusammenzuklicken.
  • Durch den Einsatz von Stylesheets ("CSS") und durch die Verwendung eigener Bilder und Icons werden vollkommen individuelle Designs möglich.
  • Mit fortgeschritteneren HTML/CSS Kenntnissen können eigene Widgets definiert werden (Es kommt die Template-Sprache "EJS" zum Einsatz).
  • wer komplexe Widgets mit eigener Logik, Interaktionen, Animationen u.Ä. erstellen will muss sich mit den verwendeten Bilbiotheken und den Konzepten des DashUI vertraut machen.

DashUI baut auf jQuery, dem jqHomematic Plugin und CanJS auf und macht sich das jQuery UI Framework zu nutze. Darüberhinaus sind diverse kleinere jQuery Plugins und Javascript-Bibliotheken für verschiedene UI-Elemente integriert.

Es ist geplant DashUI auch auf andere Backends als die HomeMatic CCU aufzusetzen, zu einem späteren Zeitpunkt soll DashUI auch mit OpenHAB und IP-Symcon zusammenspielen.

Weitere Informationen zum DashUI-Projekt finden sich in der Präsentation des DashUI vom Homematic-Usertreffen in Sottrum am 27.4.2013 und in diesem Foren-Thread.

Schnellstart
  • Es wird ein Zugang zur CCU via FTP oder SCP benötigt.
  • Addon "WebAPI" installieren: den Ordner "webapi" auf diesem Zip-File https://github.com/hobbyquaker/WebAPI/archive/master.zip nach /www/addons/ kopieren.
  • Den Ordner "dashui" aus diesem Zip-File https://github.com/hobbyquaker/DashUI/archive/master.zip ebenfalls nach /www/addons/ kopieren
  • auf der CCU leeren Ordner "dashui" in /usr/local/addons/ erstellen (hier werden Konfigurationen gespeichert)
  • Mit einem modernen Browser (Chrome, Safari, Firefox) http://ccu/addons/dashui/?edit aufrufen
  • Nun können Widgets und Views hinzugefügt und konfiguriert werden, hier ein einfaches Beispiel um eine Temperatur anzuzeigen:
    • Auf den Reiter "Widgets" wechseln
    • Im Widget-Set "basic" das Widget "hm_val - Number auswählen
    • "Add Widget" klicken. Es erscheint das gestrichelt umrandete Widget und im Reiter Widgets werden weitere Eingabefelder ("Widget-Attributes") sichtbar
    • Im Eingabefeld "hm_id" die Adresse (z.B. BidCos-RF.EEQ0001234:1.TEMPERATURE) oder ID eines Homematic-Temperature-Datenpunktes eintragen
    • In das Eingabefeld "html_append" °C eintragen. Weitere Konfigurationsmöglichkeiten siehe dieser Screenshot.
  • Widgets werden automatisch aktualisiert sobald der Editor geschlossen wird
  • Man muss manuell über die Buttons "save views on ccu" im Reiter Homematic die Konfiguration auf der CCU sichern. Automatisch gesichert wird nur im "LocalStorage" des Browsers.

FAQ
Wo finde ich Infos zu den Widget-Attributen
Die einzelnen Widgets werden im Wiki dokumentiert
Wo kann ich mehr über die Funktion der verschiedenen CSS-Attribute erfahren?
Eine gute Informationsquelle ist das "Selfhtml"
wie kann man ein Hintergrundbild einbinden?
Das geht (sowohl für Views selbst als auch für Widgets) über die CSS Eigenschaft "Background". Du kannst Dir die Bilder z.B. auf der CCU unter /www/addons/dashui/user/ ablegen und es dann mit url(user/bild.png) no-repeat als Widget-Hintergrund anzeigen lassen.
kann man einem Widget ein eigenes Bild geben? wenn ja, wie?
Entweder wie oben beschrieben über die CSS-Background-Eigenschaft oder als HTML Element (dafür ist die Widget-Eigenschaft html_prepend z.B. gedacht). So kannst z.B. mit dem Widget "basic - hm_val Number" einfach eine Temperaturanzeige mit Thermometer-Icon realisieren:
  • factor: 1
  • digits: 1
  • html_prepend: <img src="user/thermometer_icon.png" width="32px"/>
  • html_append: °C

Widget-Sets

Die Widgets sind in verschiedenen Sets gruppiert:

basic
Einfache Widgets ohne vorgegebenes Styling
fancyswitch
Eine Sammlung von hübsch gestalteten Ein/Aus-Schaltern
jqui
Widgets im jQuery UI style
jqui-mfd
Widgets im jQuery UI style und mit vordefinierten Icons aus dem mfd-icon-set
dev
Widgets für Entwickler und zur Fehlerdiagnose

Widget-Typen

Jedes Widget ist einem der folgenden Typen zugeordnet:

static
Anzeige von statischen Inhalten (z.B. ein Text, ein Bild, ein iFrame, ...)
hm_val
Anzeige von Homematic-Werten
hm_ctrl
Homematic steuern
navigation
Widgets zur Navigation zwischen den Views
container
Widgets die ganze Views innerhalb einer anderen View anzeigen

Allgemeine Widget-Attribute

html_prepend und html_append

Diese bei vielen Widgets vorhandenen Attribute dienen dazu beliebigen HTML-Quellcode vor und hinter dem eigentlich Widget-Inhalt zu platzieren.

hm_id und hm_wid

Das Attribut "hm_id" muss bei jedem Widget vom Typ "hm_val" und "hm_ctrl" angegeben werden, also allen Widgets die mit der Homematic CCU interagieren. Zum nachschauen dieser IDs bietet sich das CCU-Addon HQ WebUI an. Vorsicht, falsche IDs können die Stabilität der CCU beeinträchtigen. Hinweis: Die ID 65535 dient als Platzhalter und wird bei der Kommunikation mit der CCU ausgespart.

  • hm_id ist die ID eines Datenpunkts (STATE, LEVEL, TEMPERATURE, ...).
  • hm_wid (kann weggelassen werden) ist die ID des zugehörigen WORKING Datenpunkts, sinnvoll bei Dimmern und Rollläden um springende Slider während Aktivität der Aktoren zu verhindern.

Anstelle von IDs können auch Datenpunkt-Bezeichner wie z.B. "BidCos-RF.EEQ00012345:1.LEVEL" oder bei Programmen und Systemvariablen deren Namen verwendet werden.

externe Steuerung

DashUI legt auf Eurer CCU für jede Instanz - d.h. für jeden Browser mit dem Ihr DashUI aufruft - 2 Variablen an, dashui_instanz_cmd und dashui_instanz_data. Damit ist es möglich DashUI mit der CCU zu steuern. Man kann so z.B: einen bestimmten Browser veranlassen die View zu wechseln indem man in die Variable ..._data den Namen einer View und in die Variable ..._cmd "changeView" eingibt (per Programm, Script, ...). Sinnvoll ist das z.B. um den Browser eines bestimmten Tablets dazu zu veranlassen automatisch auf die View mit der Haustür-Webcam umzuschalten wenn die Türklingel betätigt wird
